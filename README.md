  <h2>âš™ï¸ Technical Implementation Details</h2>
  
  <table>
    <tr>
      <td>
        <h3>Database Schema</h3>
        <p>The application uses a relational database with the following main entities:</p>
        <ul>
          <li><strong>User</strong>: Stores user authentication information and preferences</li>
          <li><strong>Category</strong>: Defines expense categories with names and colors</li>
          <li><strong>Expense</strong>: Records individual expenses with amounts, descriptions, dates, and references to categories</li>
          <li><strong>Budget</strong>: Defines spending limits for categories over specific time periods</li>
          <li><strong>Receipt</strong>: Manages image references for expense documentation</li>
        </ul>
        <p>Relationships between entities are maintained through foreign keys with cascading deletions to ensure data integrity.</p>
      </td>
    </tr>
    <tr>
      <td>
        <h3>Authentication Flow</h3>
        <p>The app implements a secure authentication process:</p>
        <ol>
          <li>User registers with email and password</li>
          <li>Firebase Authentication validates credentials and creates user account</li>
          <li>Email verification is sent to confirm user identity</li>
          <li>Login process verifies credentials against Firebase</li>
          <li>Authentication state is maintained across app sessions</li>
          <li>Secure password reset functionality available</li>
        </ol>
      </td>
    </tr>
    <tr>
      <td>
        <h3>Performance Optimizations</h3>
        <p>Several techniques are employed to ensure optimal app performance:</p>
        <ul>
          <li>Pagination for large expense lists to minimize memory usage</li>
          <li>Image compression for receipt photos to reduce storage requirements</li>
          <li>Efficient SQL queries with appropriate indices</li>
          <li>Background processing for database operations</li>
          <li>UI rendering optimizations with RecyclerView and view recycling</li>
          <li>Lazy loading of images and content</li>
        </ul>
      </td>
    </tr>
  </table>
